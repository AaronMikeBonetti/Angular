Routing

Angular routing is a critical feature of the Angular framework that allows developers to build single-page applications (SPAs) and manage navigation between different views and components. It enables users to interact with the application without the need for full page reloads, providing a more seamless and responsive user experience. While this can be done in the app.module.ts file, I like to use a dedicated routing module unsurprising called app-routing.module.ts.

app-routing.module

In this file we first want to import all the dependencies so the router can have access to them, then we can simply create a routing array of route objects which include what the route is pointing to as well as what component we would like to render. Then we use the forRoot method to create and configures a module with all the router providers and directives(See Directives here) Finally we export the routes module so it can be exposed to the rest of the application.

Routing in the Template

Once we have our routing module set up we can now use them in the UI to navigate through the application. For this example Im using a nav tag but if you noticed in this project I have a dedicated navbar component where my routes are located, You can use your routes anywhere you like however. We use the routerLink directive to tell angular you want to use a route when that hyperlink is clicked.

You'll also notice in that example I'm using routerLinkActive and router-outlet. routerLinkActive just tells angular that when that route is clicked or active, we want it to have an active style or state like an underline. The router-outlet is an Angular directive from the router library that is used to insert the component matched by routes to be displayed on the screen. Simply put its the place where we bootstrap or put the specific routes component that is being routed to. Most likely you will only include this in the app.component.html file.
